## Authentication Notes

- Institution endpoints use bearer authentication through `institutionToken`.
- The OAuth token exchange endpoint relies on the client assertion payload instead of a bearer token.
- `Import Card` requires `x-api-key`.
- `Get Institution Permissions`, `Get Banks`, and `Name Enquiry` are treated as public endpoints in these docs.

## Request Patterns

- Most write operations send `application/json` payloads with example bodies already attached to the request.
- Card-related security operations use nested encrypted objects such as `encryptedData` and `publicKeyFingerprint`.
- List endpoints expose optional query filters, and those filters are available in the API playground.

## Known Coverage Gaps

The following requests need confirmed route details before they can be added as live API playground pages:

- `List Institution Transactions`
- `List Institution Users`
- `Generate RSA Key`
